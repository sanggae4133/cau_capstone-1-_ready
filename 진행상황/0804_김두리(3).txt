///스프핑 빈과 의존관계/////

굳이 여러개의 객체를 생성할 필요가 없을땐 매번 new로 객체를 생성하기보다는 스프링컨테이너한테 등록후 계속 가져다 쓰는 식으로 한다.
생성자에 @Autowired -> 스프링컨테이너에서 연관된 객체를 넣어줌.
스프링컨테이너에 객체를 등록할때엔 @Service가 필요

컨트롤을 통해서 외부요청을 받고 서비스를 비즈니스로직 레포지토리에서 데이터저장

이렇게 등록된 객체를 Autowired를 통해서 연결한다 -> DI

스프링 빈을 등록하는 2가지방법(정형화된 코드에 쓰임)
1.-@을 이용한것 -> 컴포넌트 스캔과 자동 의존관계
실행시키는 패키지만 스프링빈으로 등록가능
생성자주입 >>setter주입>>필드주입


2.자바 코드로 직접 스프링 빈 등록(상황에따라 구현 클래스를 변경해야 할 때)
@Configuration -> @Bean 이용
Controller는 그대로 한다.

-> 스프링 빈에 등록되지 않았다면 Autowired 불가

///웹 mvc 개발//
웹 브라우저가 요청을 보내면 관련컨트롤러를 찾고 그다음에 static을 찾아봄
그렇기떄문에 mapping에서 관련된 컨트롤러를 실행하기 떄문에 만들어놨던 static의 문서를 실행되지 않음.

form action을 통해 값을 입력받으면 post방식으로 넘어옴 -> postmapping으로 들어감(데이터를 폼으로 전달할때)
(url을 치는 것은 기본적으로 get방식-조회할때)
